# 图书管理系统 (Library Management System)

本项目是一个基于 Web 的全功能图书管理解决方案。系统通过迭代开发，逐步实现从基础借阅到高级自动化管理的完整流程。

## 🛠 技术栈
* **前端**: React.js
* **后端**: Node.js
* **数据库**: SQLite

---

## 📅 迭代开发路线图 (Product Backlog)

### 第一阶段 (Iteration R1)：核心基础与基础流通
**目标**：建立用户系统与核心图书流转逻辑。
* **用户管理**：读者注册、登录/登出；管理员维护读者账户。
* **图书维护**：图书管理员进行书籍的增删改查。
* **基础流通**：读者搜索书籍并提交借还请求；图书管理员审批请求并更新库存。
* **系统安全**：管理员确保数据一致性（防重复借阅、库存同步）。

### 第二阶段 (Iteration R2)：规则保障与增强体验
**目标**：引入押金机制、预约功能及通知系统。
* **规则设定**：管理员设置借阅规则与押金要求；读者支付押金。
* **流程控制**：在批准借书前强制验证押金状态与系统约束。
* **进阶流通**：读者可续借及预约书籍；管理员管理预约分配。
* **个人追踪**：读者查看借阅记录、到期时间，并接收逾期提醒通知。

### 第三阶段 (Iteration R3)：自动化处罚与互动评价
**目标**：实现自动化财务管理及社交化图书评价。
* **自动化罚款**：系统自动生成逾期罚款；管理员审核并处理罚款；读者查看并缴纳罚金。
* **权限限制**：管理员限制未缴清罚款读者的操作权限。
* **UI/UX 优化**：提供更简洁、响应式的交互界面。
* **社交功能**：读者对书籍进行评分、撰写评论；管理员管理评分与评论内容。
* **推荐系统**：基于评分展示热门书籍，并为读者推荐高质量读物。

## 👥 角色功能划分

| 角色 | 核心职责 |
| :--- | :--- |
| **读者 (Reader)** | 账号注册、图书搜索、支付押金、借还申请、续借预约、接收通知、缴纳罚金、评分评论。 |
| **图书管理员 (Librarian)** | 书籍库存管理、读者账户管理、审批借还/预约、维护借阅记录、审核罚款、管理评论。 |
| **系统管理员 (Admin)** | 设定系统规则/约束、确保数据一致性、自动化罚款策略生成、权限管控、系统监控。 |
