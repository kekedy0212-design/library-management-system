## 🚀 主要改进

### 1. 日志格式增强

#### 日志级别说明
| 级别 | 符号 | 用途 | 示例 |
|------|------|------|------|
| INFO | ✅ | 成功完成的操作 | `✅ [登录成功]` |
| WARNING | ⚠️ | 需要注意的情况 | `⚠️ [登录失败]` |
| ERROR | ❌ | 系统错误 | `❌ [数据库异常]` |
| DEBUG | 📌 | 调试信息 | `📌 [CRUD] 查询结果` |

#### 日志格式规范
```
[时间] [级别] [模块]::[函数]::[行号] - [符号] [操作分类] 操作描述 | 详细信息
```

**例子**:
```
2026-04-14 18:38:43 [INFO] app.main:init_admin:48 - ✅ [管理员] 已创建默认系统管理员账号
2026-04-14 18:38:44 [INFO] app.api.v1.endpoints.books:create_book:45 - ✅ [书籍创建成功] 用户 'admin' 创建新书籍 | 书名: '高效能人士的七个习惯' | ISBN: 9787111230113 | ID: 1
```

### 2. API 请求日志

#### 请求日志中间件增强

现在每个HTTP请求都会记录：
- ✅ 请求方法和路径
- ⏱️ 执行时间（毫秒）
- 📊 响应状态码
- 👥 客户端IP

**示例**:
```
✅ [200] GET /api/v1/books | 客户端: 127.0.0.1 | ⏱️ 0.023s
❌ [404] GET /api/v1/books/999 | 客户端: 127.0.0.1 | ⏱️ 0.015s
⚠️ [403] POST /api/v1/admin/logs | 客户端: 127.0.0.1 | ⏱️ 0.008s
```

#### 状态码符号对应
- ✅ 2xx (成功)
- ↪️ 3xx (重定向)
- ⚠️ 4xx (客户端错误)
- ❌ 5xx (服务器错误)

## 📊 日志级别指南

### INFO (默认级别)
用于记录系统正常运行的重要事件

```python
logger.info(f"✅ [登录成功] 用户 '{username}' 成功登录")
logger.info(f"✅ [书籍创建] 创建书籍 {book.title} (ID: {book.id})")
```

### WARNING
用于记录需要关注但不是错误的情况

```python
logger.warning(f"⚠️ [登录失败] 用户名或密码错误: {username}")
logger.warning(f"⚠️ [库存不足] 书籍 {book.title} 无可用副本")
```

### ERROR
用于记录系统错误

```python
logger.error(f"❌ [数据库异常] {str(e)}")
logger.error(f"❌ [文件读取失败] {str(e)}")
```

### DEBUG
用于开发调试，包含细节信息

```python
logger.debug(f"📌 [CRUD] 获取书籍: {book.title} (ID: {book_id})")
logger.debug(f"📌 [认证] 用户认证成功 | 用户名: {username} | 角色: {user.role}")
```

## 🔧 使用环境变量

### 设置日志级别
```bash
# 启用 DEBUG 日志
LOG_LEVEL=DEBUG uvicorn app.main:app --reload

# 保持默认 INFO 级别
uvicorn app.main:app --reload
```

### 自定义日志目录
```bash
# 指定自定义日志目录
LOG_DIR=/var/log/library python app.main:app
```

## 💡 最佳实践

### 1. 日志分类规范
使用中括号标记不同的操作类型：
```python
# ❌ 不好
logger.info("User logged in")

# ✅ 好
logger.info(f"✅ [登录成功] 用户 '{username}' 成功登录 | 角色: {role}")
```

### 2. 包含关键信息
```python
# ❌ 不好
logger.info("Book created")

# ✅ 好
logger.info(f"✅ [书籍创建成功] 创建书籍 | 书名: {title} | ISBN: {isbn} | ID: {book_id}")
```

### 3. DEBUG vs INFO
```python
# INFO 级别 - 业务关键事件
logger.info(f"✅ [用户认证] 用户 {username} 认证成功")

# DEBUG 级别 - 执行细节
logger.debug(f"📌 [数据库查询] SQL: SELECT * FROM users WHERE id = {user_id}")
```

### 4. 错误日志配对
```python
# 记录失败原因
if not book:
    logger.warning(f"⚠️ [查询失败] 书籍未找到 | ID: {book_id}")
    raise HTTPException(status_code=404, detail="Book not found")
```

## 📈 性能考虑

- 日志输出采用异步操作，不阻塞主线程
- RotatingFileHandler 自动轮换日志文件（10MB 触发）
- 保留最近 5 个日志备份
- DEBUG 日志只在需要时启用

## 📞 故障排查

### 问题：日志文件不生成
- ✅ 检查 `logs/` 目录是否存在（自动创建）
- ✅ 检查日志级别是否为 INFO 或以下
- ✅ 检查文件权限是否正确

### 问题：日志输出到控制台为乱码
- ✅ 这是编码问题，不影响文件中的日志
- ✅ 文件中日志显示正常

### 问题：日志文件过大
- ✅ 系统会自动轮换（每 10MB 一次）
- ✅ 保留最近 5 个备份
